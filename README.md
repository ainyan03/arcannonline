# blt — ブラウザ P2P 弾幕フィールドゲーム（試作）

オープンフィールドをキャラクターが歩き回り、弾幕で撃ち合う多人数参加型ブラウザゲームの試作。
ゲームサーバを持たず、近距離のプレイヤー同士が WebRTC DataChannel で直接同期する。

## アーキテクチャ

```
ブラウザA ──── WebSocket ────┐
                             ├─ シグナリングサーバ（接続情報と座標のみ保持。
ブラウザB ──── WebSocket ────┘   近傍の出入り通知と WebRTC シグナリング中継）
    │
    └───── WebRTC DataChannel（P2P直接通信） ───── ブラウザB
            ├ state    チャネル: 位置・向き（非信頼・非順序、~15Hz）
            └ reliable チャネル: チャット・弾幕発射イベント（予定）
```

- 演算は 2D 平面 (x, y)。描画時に XZ 平面へ写像し Three.js で 3D 俯瞰表示する。
- 近傍管理は二層構造: 低頻度（1秒毎）の座標報告をサーバへ送り、サーバが
  「60 units 以内で接続・80 units 超で切断」のヒステリシス付きで出入りを通知する。
- 高頻度な位置同期はサーバを経由せず P2P で行う。
- NAT 越えは STUN のみ（TURN なし）。直接通信が確立できない相手とは同期しない割り切り。
- 被弾判定は自己申告制（チート対策はスコープ外）。

## 起動方法

```bash
npm install
npm run dev:server   # シグナリングサーバ (ws://localhost:8787)
npm run dev:client   # Vite 開発サーバ (http://localhost:5173)
```

ブラウザ（タブ）を2つ以上開いて http://localhost:5173 に参加すると、互いのキャラクターが見える。
LAN 内の他端末からは `http://<ホストのIP>:5173` でアクセスできる。

## 操作方法

- WASD / 矢印キー: 移動（カメラ基準）
- マウスドラッグ: 視点回転
- ホイール: ズーム

## ディレクトリ構成

```
shared/src/protocol.ts   共通プロトコル定義・定数
server/src/index.ts      シグナリングサーバ (Node.js + ws)
client/src/
  net/                   シグナリング・WebRTC ピア管理
  game/                  描画・入力・プレイヤー
  ui/                    参加オーバーレイ
```
